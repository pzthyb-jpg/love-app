## RED-1：打卡路径 6 步太长

### 1.1 现状

当前路径：  
`底部 Tab 拍照` → `点进 Photo 页` → `点击「咔嚓拍照」` → `等待摄像头打开` → `再次点击「咔嚓！」` → `点击确认「用这张」` → `Toast 提示`

共 **6 次用户操作**，且被摄像头权限阻塞。

### 1.2 目标

提供两条路径：
- **拍照打卡**：保留现有 6 步流程（照片是核心价值）
- **快速打卡**：首页「立即打卡」按钮 → Toast 反馈 = **2 步**

### 1.3 功能点

| 编号 | 功能 | 描述 |
|------|------|------|
| F1.1 | 首页「立即打卡」按钮 | 首页 `love-data-card` 内新增按钮；点击后直接写入一条无照片的打卡记录 |
| F1.2 | 今日状态联动 | 按钮点击后今日状态圆圈从「未打卡」变为「已打卡」（与拍照打卡效果一致） |
| F1.3 | 数据兼容 | 后端数据格式不变：checkin record 的 `photo` 字段为可选，`type: 'quick'` |
| F1.4 | 视觉区分 | 有照片的打卡 vs 无照片的打卡在「本周打卡」区域使用不同图标（实心 ⚪ / 空心 🔘） |
| F1.5 | 按钮状态 | 已打卡后按钮置灰 + 文案变为「今日已打卡」，不可重复点击 |

### 1.4 交互细节

1. 按钮位置：首页「爱的数据」卡片内，计数器下方、「本周打卡」区域上方
2. 按钮样式：Vant `van-button type="primary" size="small"`，宽度自适应
3. 点击逻辑：
   - 未打卡 → `store.addCheckin({ date: today, type: 'quick' })` → 按钮禁用 → `showToast('✅ 已打卡')`
   - 已打卡 → 按钮 disabled，点击无响应
4. 字符串常量：`'quick-checkin-btn'`、`'checked-in-today'`

### 1.5 验收标准

- [ ] AC-1.1：未打卡用户首页可见「立即打卡」按钮，点击后按钮变灰
- [ ] AC-1.2：点击按钮后 store 中 `checkinHistory` 新增一条 `{ date, type: 'quick' }` 记录
- [ ] AC-1.3：今日状态区域打卡圆圈从空心变为实心，文案从「未打卡」变「已打卡」
- [ ] AC-1.4：同一日重复点击按钮不产生重复记录
- [ ] AC-1.5：拍照打卡记录包含 `{ date, photo, type: 'photo' }`，与快速打卡数据结构兼容
- [ ] AC-1.6：从用户决定打卡到完成反馈 ≤ 3 秒（无摄像头等待）

## RED-2：漂浮爱心不可关闭

### 2.1 现状

`Home.vue` 有 `.floating-hearts` 动画容器，`v-for="i in 6"` 渲染 6 颗漂浮爱心，CSS `@keyframes floatUp` 控制从底部漂浮到顶部。无任何 UI 开关。

### 2.2 目标

设置页新增「动画密度」选项，4 档位：

| 档位 | 值 | 行为 |
|------|-----|------|
| 关闭 | `off` | 不渲染 `.floating-hearts`，无动画 |
| 紧凑 | `compact` | 渲染 3 颗爱心，速度 -50%，透明度 -30% |
| 完整 | `full` | 当前默认：6 颗爱心，原速度 |
| 密集 | `density` | 12 颗爱心，速度 +30% |

### 2.3 功能点

| 编号 | 功能 | 描述 |
|------|------|------|
| F2.1 | 设置页新增「动画密度」Cell | 位于「主题设置」卡片，「Dark mode toggle」下方 |
| F2.2 | 4 档选择器 | 使用 `van-radio-group` 横向排列，或 `van-dropdown-menu` |
| F2.3 | 实时预览 | 选择后首页动画立即生效（无需刷新） |
| F2.4 | localStorage 持久化 | 与 theme_prefs 同级存储 `animation_density` |
| F2.5 | CSS 变量驱动 | 通过 `data-animation` 属性控制 CSS 规则 |

### 2.4 交互细节

1. 设置页 Cell 结构：左侧图标 🫧 + 右侧 `van-dropdown-menu` 或行内 `van-radio-group`
2. 数据存储：`localStorage.setItem('animation_density', value)`
3. Home 挂载时读取：`document.documentElement.dataset.animation = storedValue`
4. CSS 规则：
   ```css
   [data-animation="off"] .floating-hearts { display: none; }
   [data-animation="compact"] .floating-hearts { --heart-count: 3; --opacity-mult: 0.5; }
   [data-animation="full"]    .floating-hearts { --heart-count: 6; }
   [data-animation="density"]  .floating-hearts { --heart-count: 12; --speed-mult: 1.3; }
   ```

### 2.5 验收标准

- [ ] AC-2.1：设置页「动画密度」Cell 可见，包含 4 个选项
- [ ] AC-2.2：选择「关闭」后进入首页，无任何漂浮爱心渲染
- [ ] AC-2.3：选择「紧凑」后首页显示 3 颗爱心，动画放慢
- [ ] AC-2.4：选择「密集」后首页显示 12 颗爱心
- [ ] AC-2.5：刷新页面后设置仍然保持（localStorage 验证）
- [ ] AC-2.6：修改设置后返回首页即时生效，无需手动刷新

## RED-3：纪念日入口 ×3 混乱

### 3.1 现状

用户可通过 3 个入口管理纪念日：

| 入口 | 位置 | 行为 |
|------|------|------|
| 首页 `.love-days-empty` 卡片 | Home.vue | 点击弹出 Dialog（选择日期 + 保存）|
| 设置页「纪念日」Cell | Settings.vue | 点击打开 `van-date-picker` |
| 底部「纪念」Tab | Anniversary.vue | 独立纪念日管理页（已有完整 CRUD）|

### 3.2 目标

统一为单一入口模型：

| 位置 | 职责 | 行为 |
|------|------|------|
| 首页 | **展示** | 倒计时数字 + 天数（只读展示，不可点击编辑）|
| 设置页 | **引导** | Cell 文案「管理纪念日 ›」，点击跳转纪念Tab |
| 纪念Tab | **管理** | 完整 CRUD（保持不变）|

### 3.3 功能点

| 编号 | 功能 | 描述 |
|------|------|------|
| F3.1 | 首页倒计时只读化 | 移除 `.love-days-empty` 的 `@click` 和整个 Dialog |
| F3.2 | 首页有纪念日的卡片只读 | 即使已设置纪念日，首页卡片也不可点击 |
| F3.3 | 设置页入口引导化 | 「纪念日」Cell 右侧文案改为「管理 ›」，点击跳转 `/anniversary` |
| F3.4 | 空态引导 | 未设置纪念日时，首页卡片文案：「去记录我们的第一天 →」，点击跳转纪念Tab |
| F3.5 | 设置页删除独立日期选择器 | 移除设置页的 `van-date-picker` 和 `van-popup` 相关代码 |

### 3.4 交互细节

1. 首页纯展示（无论是否已设置纪念日），点击无响应
2. 设置页已有 `router.push('/settings')` 模式，直接使用 `router.push('/anniversary')`
3. 设置页移除日期选择器后，对应 `van-popup` / `van-date-picker` 组件和 JS 逻辑同步清理
4. 文案「管理纪念日」使用 `setting-label`，右侧箭头使用现有 `setting-arrow` 类

### 3.5 验收标准

- [ ] AC-3.1：首页倒计时卡片点击无响应（不弹 Dialog，不打开选择器）
- [ ] AC-3.2：设置页「纪念日」Cell 点击后跳转到纪念Tab（路由变化可见）
- [ ] AC-3.3：设置页不再渲染 `van-popup` / `van-date-picker` 组件
- [ ] AC-3.4：未设置纪念日时首页显示引导文案，点击跳转纪念Tab
- [ ] AC-3.5：纪念日管理的唯一功能入口是纪念Tab

## 5. 全局验收（对抗性回归）

修复完成后，用以下场景做端到端回归：

1. **大强首次打卡**：打开 App → 0.5s 内看到「立即打卡」→ 1s 内点击 → 0.5s 内看到反馈 → 总耗时 ≤ 2s ✅
2. **大强午休打开**：首页「已选」字样无冒犯 → 动画紧凑档无干扰 → 纪念日倒计时清晰可读 ✅
3. **大强老婆看到**：保留足够浪漫元素（滑入动画、天数大字、纪念日提醒）→ 老婆满意 ✅
4. **大强改设置**：深色/浅色措辞与系统一致 → 隐私说明读起来像正常产品 → 无幼稚感 ✅
