## 一、背景 & 目标

当前「转圈」交互为：收藏餐厅 → Canvas 转盘旋转 → 随机选中。

目标：全面升级为**台球骰子弹射**交互 —— 用户在绿色台球桌面上摇晃骰子，骰子最终点数（1-6）对应选中的餐厅，替代原有转盘玩法。

## 二、功能变更概览

### 2.1 保留不变
- 附近 Tab：POI 列表浏览、搜索、分类筛选
- 收藏（♥）/ 取消收藏逻辑
- Supabase 存储（restaurant_prefs 表、lunch_history 记录）
- 本周/TOP3 统计

### 2.2 核心变更

| 旧交互 | 新交互 |
|--------|--------|
| Canvas 圆形转盘 | 3D 绿色台球桌面 + 1-6 号骰子 |
| "旋转"按钮 | 摇晃手机触发 |
| 旋转动画停止后显示结果 | 陀螺仪检测到停止摇晃后定住骰子 |
| 结果显示在 Dialog | 直接展示在桌面上 + 底部抽屉 |

### 2.3 新功能点

#### A. 餐厅选择（最多 6 家，支持拖拽排序）
- 在「转圈」Tab，下方出现从下向上的抽屉
- 抽屉默认可见（高度约 180px），上滑查看更多
- 抽屉内横向展示 1-6 个餐厅卡片
- 用户长按餐厅卡片可上下拖拽调整顺序
- 拖动时：卡片放大 + 阴影加深，其他卡片让位
- 位置 1-6 对应骰子点数 1-6
- 超过 6 家：限制只允许选择前 6 个（最早收藏的优先）

#### B. 3D 台球桌面 + 骰子
- **桌面**：长方形绿色「台面」，有木质边框（棕色），类似台球桌
- **骰子**：3D 立方体，每面标有 1-6 点数
  - 使用 CSS 3D transforms 或 Three.js 渲染
  - 骰子六面材质：白色底 + 黑色点数（经典骰子外观）
  - 角部有圆角处理
- **台面细节**：四周有 6 个球袋（圆形凹陷），桌面有轻微纹理

#### C. 摇晃检测（陀螺仪/加速度计）
- 使用 `DeviceMotionEvent` 监听手机加速度
- 算法：
  1. 用户按住/点击「掷骰子」按钮后开始监听
  2. 计算加速度变化率（相邻两次事件的 |Δx|+|Δy|+|Δz| 之和）
  3. 若变化率 > 阈值，判定为"摇晃中"
  4. 若变化率连续 800ms 低于阈值，判定为"停止"
  5. 停止时骰子快速减速并定住
- **防误触**：只有明确按住骰子 / 点击按钮后才进入"准备抛掷"状态
- **冷却**：定住后 2 秒内不可再次触发

#### D. 骰子掷出动画
1. **蓄力阶段**：用户按住按钮，骰子上下抖动（蓄力感）
2. **抛掷阶段**：用户松手（或摇晃检测触发），骰子弹射到桌面
   - 初速度方向随机反弹
   - 在台面上滚动/弹跳 3-5 次
3. **减速阶段**：摩擦力模拟，速度逐渐降低
4. **定格阶段**：骰子最终停在某个朝向
   - 顶部面朝上的数字 = 选中结果

#### E. 结果展示 & 记录
- 骰子定格后 500ms，桌面高亮显示选中餐厅
- 同时 Toast: "🎲 结果是: [餐厅名]（点数 N）"
- 自动写入 lunch_history（同现有逻辑）
- 底部抽屉对应序号的卡片有发光动画

#### F. 兜底方案（无陀螺仪设备）
- 检测 `DeviceMotionEvent` 是否可用
- 若不可用：点击骰子 → 直接触发随机结果 + 3D 旋转动画
- 保留原有按钮交互

## 三、UI / UX 规范

### 3.1 底部抽屉
- **高度**：收起 60px / 展开 200px
- **视觉**：白色圆角卡片（顶部圆角 16px），带顶部把手指示条
- **动画**：上滑展开 300ms spring，下滑收起 250ms ease
- **位置**：固定在视口底部（在 tabbar 之上）
- **内容**：
  - 标题行：「🍽️ 我的骰子 (N/6)」
  - 横向滚动的餐厅卡片列表，每张卡片显示：序号、emoji、名称
  - 长按拖动把手图标（⠿）

### 3.2 台球桌面
- **尺寸**：宽高比 3:2，最大宽度 360px，居中
- **颜色**：
  - 台面：#2E7D32（深绿，类似台球毯）
  - 边框：#5D4037（胡桃木色）
  - 球袋：#1B1B1B
- **光照**：左上方 45° 方向柔和阴影
- **骰子**：
  - 尺寸：60×60×60 px
  - 位置：台面中央偏下（黄金分割点）

### 3.3 拖拽排序交互
- 长按 300ms 触发拖拽模式
- 被拖动卡片：scale(1.08) + box-shadow-lg + z-index: 100
- 其他卡片瞬移让位（无动画，即时对齐）
- 松手后：被拖卡片落位，所有卡片 200ms ease 归位
- 拖动中：卡片上方显示半透黑色遮罩 20%

## 四、技术方案

### 4.1 3D 骰子渲染
**方案 A（推荐）：纯 CSS 3D transforms**
- 6 个 div 分别代表骰子 6 面
- 使用 `transform-style: preserve-3d` 构建立方体
- 旋转动画：`transform: rotateX() rotateY() rotateZ()`
- 优点：零依赖、性能优、兼容性好
- 缺点：物理模拟需手写

**方案 B：Three.js + Cannon.js**
- 真实物理引擎模拟骰子弹跳
- 优点：物理效果逼真
- 缺点：包体积大 (~200KB)、复杂度高

**决策**：方案 A（CSS 3D），手绘物理轨迹，效果足够且不增加包体积。

### 4.2 陀螺仪检测
```js
// 伪代码
window.addEventListener('devicemotion', (e) => {
  const acc = e.accelerationIncludingGravity
  const delta = Math.abs(acc.x - last.x) + Math.abs(acc.y - last.y) + Math.abs(acc.z - last.z)
  if (delta > SHAKE_THRESHOLD) {
    shakeIntensity.value = delta
    isShaking.value = true
  }
})
```

### 4.3 拖拽排序
- 使用 HTML5 Drag & Drop API 或 touch events
- 推荐：`@touchstart` / `@touchmove` / `@touchend` 手动实现
- 计算当前手指位置对应的插入索引

### 4.4 新增文件
- `src/components/DiceTable.vue` — 3D 台球桌面 + 骰子组件
- `src/components/RestaurantDrawer.vue` — 底部抽屉 + 拖拽排序
- `src/composables/useShakeDetector.js` — 陀螺仪检测 composable
- 修改 `Lunch.vue` — 替换 wheel-section 为 DiceTable + RestaurantDrawer

## 六、风险 & 依赖

| 风险 | 缓解 |
|------|------|
| iOS 13+ 需要用户授权 DeviceMotion | 首次点击时调用 `DeviceMotionEvent.requestPermission()` |
| 部分安卓设备不支持陀螺仪 | 降级为点击触发 |
| CSS 3D 在低端安卓机可能卡顿 | 提供 `prefers-reduced-motion` 降级 |
| 拖拽排序在 iOS Safari 有 touch 延迟 | 使用 `touch-action: none` + passive: false |
