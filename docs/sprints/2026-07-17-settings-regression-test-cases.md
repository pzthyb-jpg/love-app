### TC-001：修改女朋友昵称

| 项目 | 内容 |
|------|------|
| **编号** | TC-001 |
| **名称** | 修改女朋友昵称 |
| **严重级** | P0 |
| **前置条件** | 1. 用户已登录（testuser）<br>2. 页面已加载完成<br>3. 女朋友昵称初始值为空或默认值 |

**操作步骤：**

1. 进入 `/settings` 页面
2. 在「女朋友昵称」输入框中输入 `小甜心`
3. 点击输入框外部区域（触发 blur）
4. 观察页面 Toast 提示
5. 刷新页面，观察昵称是否保留

**预期结果：**

- 输入框实时显示输入内容 `小甜心`
- blur 后弹出 Toast "已保存"（绿色成功样式）
- 刷新后输入框仍显示 `小甜心`
- 验证 `dataStore.state.settings.girlfriend_name` 值为 `小甜心`
- 验证 Supabase `user_settings` 表中 `girlfriend_name` 字段已更新

**验证方式：**

- [ ] UI 视觉检查
- [ ] Vue DevTools 检查 dataStore 状态
- [ ] 网络请求检查 PATCH /user_settings 返回 200
- [ ] localStorage 检查（无，因为走 Supabase）

## 5. 缺陷严重级定义

| 级别 | 定义 | 示例 |
|------|------|------|
| **P0** | 核心功能不可用，必须修复 | 登录失败、昵称无法保存、提醒无法开启 |
| **P1** | 功能可用但体验受损，或边界场景异常 | 网络错误提示不明确、超长字符未限制 |
| **P2** | 轻微 UI 问题或极端场景 | 快速点击闪烁、样式偏差 |

## 6. 风险与依赖
