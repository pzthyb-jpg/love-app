### 1.2 反向流程（Error/Edge Cases）

| 编号 | 用例名称 | 触发条件 | 用户看到的 | 系统行为 |
|------|---------|---------|-----------|---------|
| TC-R01 | 空名称保存 | 不填名称，直接保存 | 错误提示"请输入名称"（红色） | 不跳转，不发送请求 |
| TC-R02 | 无日期保存 | 填名称，不选日期 | Toast "请选择日期" | 不跳转 |
| TC-R03 | 名称超长（>20字） | 输入 21+ 字符 | maxlength=20 截断 | 只保存前 20 字符 |
| TC-R04 | 备注超长（>100字） | 输入 101+ 字符 | maxlength=100 截断 | 只保存前 100 字符 |
| TC-R05 | 编辑不存在的纪念日 | 访问 /anniversary/nonexistent | Toast "未找到该纪念日" | 自动跳回列表 |
| TC-R06 | 未登录访问 | localStorage 清除后访问 /anniversary | 跳转到 /login | 路由守卫拦截 |
| TC-R07 | 取消日期选择 | 打开日期选择器→取消 | 日期仍为空 | UI 不崩溃 |
| TC-R08 | 取消删除 | 点击删除→取消 | 数据仍在列表 | 不发送 DELETE 请求 |
| TC-R09 | 未登录添加 | 未登录时直接 POST /annabases | 返回 { error: "未登录" } | 401 错误 |
| TC-R10 | 快速连续点击保存 | 连续快速点击 3 次保存按钮 | isLoading 防止重复提交 | 只发送一次请求 |

### 1.3 数据映射专项测试（Data Mapping）

| 编号 | 用例名称 | 操作 | 预期映射 |
|------|---------|------|---------|
| TC-D01 | 创建时前端字段→DB | 添加纪念日 | `{ name, date, type, emoji, remark, remindDays:[1,3,7] }` → `{ remind_days: [1,3,7], ... }` |
| TC-D02 | 读取时 DB→前端 | 获取纪念日列表 | `{ remind_days: [3,7], created_at: "2025-01-01" }` → `{ remindDays: [3,7], createdAt: "2025-01-01" }` |
| TC-D03 | 编辑时前端字段→DB | 更新纪念日 | `{ remindDays: [1,3,7,30] }` → `{ remind_days: [1,3,7,30] }` |
| TC-D04 | 读取时 created_at 映射 | 获取纪念日 | `{ created_at: "2025-01-01T00:00:00Z" }` → `createdAt: "2025-01-01T00:00:00Z"` |
| TC-D05 | 空备注默认值 | 添加时备注为空 | `remark: ""` 保存到 DB，不保存 `null` |
| TC-D06 | 默认提醒天数 | 添加时不改提醒 | `remind_days: [1, 3, 7]`（默认值） |
| TC-D07 | remind_days 默认值- birthday | 添加生日类型 | `remind_days: [1, 3, 7]`（当前实现无类型差异） |

## 二、异常处理验证（PRD 第四章）

| 场景 | 测试方法 | 预期 |
|------|---------|------|
| 存储读取失败 | 手动破坏 localStorage JSON | safeGetJSON 返回 []，展示空态，不崩溃 |
| 日期格式非法 | Date 解析失败 | 显示 "—"，不崩溃 |
| 推送权限未授权 | Notification.permission = 'denied' | 静默跳过推送，不阻断操作 |
