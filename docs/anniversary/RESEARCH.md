# 纪念日模块 — 竞品分析

> 分析范围：Days Matter（时间管理类）、时间账单（生活记录类）、恋爱记（情侣专属类）、想你（情侣社交类）、小恩爱（情侣社区类）  
> 分析日期：2026-07-08  
> 目的：为 love-app 纪念日模块提供产品定位与差异化依据

---

## 1. 竞品概览

### 1.1 Days Matter（倒数日）

| 维度 | 说明 |
|------|------|
| **产品定位** | 通用型倒数日/纪念日管理工具；不做情侣专属，面向所有人 |
| **核心交互** | 列表展示所有倒数日，支持正数/倒数、按重要程度分组、iCloud 同步 |
| **差异化亮点** | 极简 UI、桌面小组件、事件分类标签、历史事件归档 |
| **用户痛点** | 无情侣情感元素（无昵称、无合照）；条目过多时视觉嘈杂；无推送需手动查看 |
| **截图印象** | 纯列表 + 数字，像任务管理器，不像情感记录 |

### 1.2 时间账单

| 维度 | 说明 |
|------|------|
| **产品定位** | 生活时间记录与回顾；强调「时间都花在哪了」 |
| **核心交互** | 条状时间线 + 圆形统计图；以「天」为单位的回顾视图 |
| **差异化亮点** | 像存折一样记录每一天的「情感存款」；具备反思和总结功能 |
| **用户痛点** | 上手门槛较高；界面偏工具化，情感温度不足 |
| **截图印象** | 像记账软件，没有「恋爱」的感觉 |

### 1.3 恋爱记

| 维度 | 说明 |
|------|------|
| **产品定位** | 情侣专属的「恋爱日记」；功能最全（打卡、聊天、相册、纪念日） |
| **核心交互** | 情侣共享一个账号；有聊天室和情侣相册；纪念日支持倒计时 |
| **差异化亮点** | 情侣双方数据互通；有里程碑自动推送；支持恋爱报告生成 |
| **用户痛点** | 需要双方共同注册共享码；界面广告较多；推送过于频繁打扰 |
| **截图印象** | 功能多但混乱，像 QQ 空间的情侣版 |

### 1.4 想你

| 维度 | 说明 |
|------|------|
| **产品定位** | 情侣远程互动产品；主打「虚拟同居」和「远程亲密」 |
| **核心交互** | 远程亲嘴、同城定位、纪念日提醒、情侣商城 |
| **差异化亮点** | 硬件配对（远程亲嘴器）；纪念日有精美插画卡片；推送走心文案 |
| **截图印象** | 花哨、商业化严重，需要双方时刻在线互动 |

### 1.5 小恩爱

| 维度 | 说明 |
|------|------|
| **产品定位** | 情侣社区 + 工具；泛社交属性，偏向「朋友圈情侣版」 |
| **核心交互** | 情侣之间发帖/签到；有纪念日提醒；内嵌商城和游戏 |
| **差异化亮点** | 社区属性强；纪念日提醒支持短信/微信/推送三路触达 |
| **用户痛点** | 社区内容鱼龙混杂；广告植入多；纪念日只是附属功能 |

## 3. 用户痛点综合

1. **情侣共享强迫症** — 恋爱记/想你/小恩爱都要求情侣双方注册同一个账号，否则功能大打折扣
2. **情感温度缺失** — Days Matter 像记事本，没有恋爱氛围
3. **推送打扰** — 多数 App 推送频率过高（每天/节假日/纪念日），用户易疲劳
4. **界面过于花哨** — 想你/小恩爱功能堆砌，学习成本上升
5. **数据丢失焦虑** — 纯本地存储的 App 很多，换手机即丢失
6. **纪念日类型单一** — 多数只支持「在一起纪念日」，不支持生日/节日/自定义

## 4. love-app 差异化定位

> **核心策略：轻单人驱动 + 极简情感、不给情侣共享设门槛、不打扰的氛围型工具**

| 维度 | love-app 选择 | 竞品对比 |
|------|---------------|----------|
| **情侣共享** | 不需要。一个 App 由一方管理，另一方看 | 恋爱记/想你：强制共享 |
| **纪念日类型** | 支持多种类型（周年/生日/节日/自定义），各有专属 emoji | Days Matter：纯文本；恋爱记：偏单一 |
| **推送策略** | 默认提前 3 天，用户可关闭；不骚扰 | 想你/小恩爱：多次频繁推送 |
| **设计语言** | 极简纯色 + Apple HIG，给女朋友一个人看 | 花哨、高饱和、商业感 |
| **情感目标** | 制造「仪式感」而非「倒计时焦虑」 | Days Matter：倒计时焦虑 |
| **存储架构** | 纯本地 + Pinia，无服务器依赖 | 多想你/小恩爱：强依赖云端 |
| **代码体积** | 纪念日模块 < 500 行 | 恋爱记同等功能 > 5000 行 |

---

## 5. 我们拿什么打

- **极简**：一看就懂的界面，不需要说明书
- **有温度**：每个纪念日有 emoji + 金色高亮 + 浮动爱心动画
- **不打扰**：推送策略默认保守，给用户控制感
- **无共享门槛**：不需要女生也用 App，男生设置好、女生看就完了
