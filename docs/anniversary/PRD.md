## 1. 功能定义

### 1.1 核心功能（MVP）

| 编号 | 功能名称 | 描述 | 优先级 |
|------|----------|------|--------|
| F1 | 纪念日列表 | 展示所有已添加的纪念日，按时间倒序排列（最近在前） | P0 |
| F2 | 添加纪念日 | 通过弹窗表单添加（名称、日期、类型） | P0 |
| F3 | 编辑纪念日 | 点击卡片进入编辑弹窗，可修改名称/日期/类型 | P0 |
| F4 | 删除纪念日 | 长按或滑出删除，二次确认弹窗 | P0 |
| F5 | 倒计时显示 | 距离下一个纪念日的天数 | P0 |
| F6 | 相守天数 | 每个纪念日的「已过天数」展示 | P0 |
| F7 | 推送提醒 | 纪念日来临前 N 天推送本地通知 | P1 |
| F8 | 纪念日详情 | 点击卡片进入详情页（详情暂为卡片展开） | P1 |
| F9 | 纪念日类型标签 | 周年/生日/节日/自定义，各有 emoji 区分 | P1 |
| F10 | 空态引导 | 列表为空时展示引导 UI + 快捷添加按钮 | P0 |

### 1.2 边界明确排除的功能

| 排除项 | 原因 |
|--------|------|
| 情侣共享/同步 | 不引入账号体系和后端 |
| 纪念日照片上传 | 当前版本不做，降低复杂度 |
| 纪念日重复规则（每年重复） | 初始版本先只支持单次日期 |
| 纪念日排序自定义 | 默认按时间排序，不做拖拽 |
| 纪念日分享/导出 | 不做社交功能 |
| 第三方推送（微信/短信） | 只做本地通知 API |

## 2. 数据结构

### 2.1 Anniversary 模型字段

```typescript
interface Anniversary {
  id: string          // 唯一ID，格式 `ann_${timestamp}`
  name: string        // 名称，如 "恋爱周年"
  date: string        // 原始日期，格式 YYYY-MM-DD
  type: 'anniversary' | 'birthday' | 'holiday' | 'custom'
  emoji: string       // 展示用 emoji
  remark: string      // 备注，可选
  remindDays: number[] // 提前多少天提醒，如 [1, 3, 7]
  createdAt: string   // 创建时间 ISO
  updatedAt: string   // 更新时间 ISO
}
```

## 3. 用户流程

### 3.1 添加纪念日

```
用户进入「纪念」Tab
  → 看到空态 或 列表
  → 点击右下角「+」按钮
  → 弹出添加弹窗（Vant Dialog）
  → 填写：名称、选择类型（决定 emoji）、选择日期、备注（可选）
  → 选择提醒时机（默认提前 3 天）
  → 点击「保存」
  → 列表刷新，新卡片带动画进场
  → Toast 提示「已添加 💕」
```

### 3.2 查看纪念日

```
用户进入「纪念」Tab
  → 看到纪念日列表（按下次发生时间排序）
  → 每个卡片展示：emoji、名称、日期、倒计时/已过天数
  → 点击卡片展开详情（inline 展开）
  → 展示：完整备注、创建时间、提醒设置
```

### 3.3 编辑纪念日

```
用户点击卡片右上角「编辑」图标
  → 弹出编辑弹窗（预填现有值）
  → 修改字段
  → 点击「保存」
  → 列表更新
  → Toast 提示「已更新」
```

### 3.4 删除纪念日

```
用户长按卡片 或 点击「删除」图标
  → 弹出确认弹窗（Vant Dialog）
  → 文案：「确定要删除【{name}】吗？删除后无法恢复」
  → 点击「确认删除」
  → 卡片消失动画
  → Toast 提示「已删除」
```

### 3.5 推送提醒流程图

```
App 启动时
  → 调用 checkAnniversaryNotifications()
  → 读取所有纪念日 + remindDays
  → 计算每个纪念日下次发生日期
  → 检查今天是否在 remindDays 匹配的日期
  → 匹配 → 发送 Notification API 本地推送
  → 当天已推送则不再重复（持久化「已推送」标记）
```

## 4. 异常处理

| 场景 | 处理方式 | 用户可见行为 |
|------|----------|--------------|
| 空数据（无纪念日） | 展示空态 UI | 大 emoji「🎂」+「还没有纪念日，添加第一个吧」+ CTA 按钮 |
| 同名纪念日 | 允许同名，ID 不同 | 不提示，不影响添加 |
| 重复日期（同一天不同名） | 允许 | 列表中有两条 |
| 过去日期 | 允许添加，已过天数为正数 | 卡片右侧显示「已过 N 天」 |
| 存储写入失败 | safeSetJSON 返回 false | Toast 提示「存储失败，请清理空间」|
| 存储读取失败（JSON 解析异常） | safeGetJSON 返回默认值 [] | 静默降级，展示空态 |
| 日期格式非法（手动操作 localStorage） | Date 解析返回 NaN | 显示「—」，不崩溃 |
| 推送权限未授权 | Notification.permission !== 'granted' | 静默跳过推送，不阻断操作 |

## 5. 验收标准

### 5.1 功能验收

| 编号 | 功能 | 验收断言 |
|------|------|----------|
| AC-F1 | 纪念日列表 | ① 默认展示空态 ② 添加后列表按最近发生时间排序 ③ 每个卡片展示 emoji、名称、日期、天数 |
| AC-F2 | 添加纪念日 | ① 弹窗显示 4 个字段 ② 名称必填，空值禁止保存 ③ 日期必选，默认今天 ④ 类型可切换 ⑤ 保存后列表立即显示 |
| AC-F3 | 编辑纪念日 | ① 弹窗预填现有值 ② 修改后保存即时生效 ③ 取消不保存 |
| AC-F4 | 删除纪念日 | ① 删除前弹出确认 ② 确认后列表移除该条目 ③ 取消不删除 |
| AC-F5 | 倒计时 | ① 未来日期显示「还有 N 天」② 今天显示「就是今天 🎉」③ 过去日期显示「已过 N 天」|
| AC-F6 | 相守天数 | 展示从 date 到今天的总天数 |
| AC-F7 | 推送提醒 | ① 提前 N 天匹配时发送推送 ② 同天不重复推送 ③ 关闭设置后不推送 |
| AC-F8 | 空态 | 列表为空时展示空态 UI，点击 CTA 打开添加弹窗 |
| AC-F9 | 类型 emoji | 不同类型显示不同 emoji 和颜色 |
| AC-F10 | 持久化 | 刷新页面后纪念日数据不丢失 |

### 5.2 技术验收

| 断言 | 验证方式 |
|------|----------|
| 代码行数 < 500 行 | cloc 统计 View + Composable + 独立 CSS |
| 复用现有 Vant 4 组件 | 不使用新 UI 库 |
| Pinia store 新增 | dataStore 新增 anniversaries 状态和操作函数 |
| useStorage 新增 KEY_ANNIVERSARIES | 常量名 KEY_ANNIVERSARIES = 'anniversaries' |
| 不影响现有功能 | 增量开发，不修改其他 Tab 代码 |
| 兼容深色模式 | 使用 CSS 变量，不写死色值 |

## 6. 推送策略（默认值）

| 纪念类型 | 默认提前天数 | 可自定义 |
|----------|-------------|----------|
| anniversary（周年） | [1, 3, 7] | 允许 |
| birthday（生日） | [1, 7, 30] | 允许 |
| holiday（节日） | [1, 3, 7] | 允许 |
| custom（自定义） | [1, 3] | 允许 |

> **推送原则**：默认保守。每个纪念日最多推送 3 次，且同一天不推送两条。推送仅使用 Notification API（PWA），不引入第三方。
