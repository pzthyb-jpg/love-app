## 一、现状分析（Before v2.0）

### 1.1 代码架构快照

| 模块 | 文件 | 行数 | 核心能力 |
|------|------|------|----------|
| 彩虹屁引擎 | `js/compliments.js` | 216 行 | 300+ 彩虹屁模板、天气/节日检测、打卡历史 CRUD |
| 拍照打卡 | `js/photo.js` | 305 行 | 摄像头调用、镜像翻转、拍照 + 彩虹屁流程、通知开关 |
| 午餐转盘 | `js/lunch.js` | 390 行 | Canvas 转盘绘制、旋转动画、餐厅 CRUD、转盘结果弹窗 |
| 愿望池 | `js/wish.js` | 168 行 | 愿望/吐槽提交、气泡展示、字符计数、单项/批量删除 |
| 路由导航 | `js/app.js` | 90 行 | Tab 切换、页面渲染调度、结果遮罩层关闭 |
| 样式系统 | `css/style.css` | 742 行 | CSS 变量体系、Za Bank 粉紫风格、卡片/按钮/转盘/气泡组件 |
| 入口文件 | `index.html` | 125 行 | SPA 骨架、Meta/PWA 配置、模块加载顺序 |

### 1.2 技术栈特点

- ✅ **零依赖** — 纯 HTML5 + CSS3 + JS，无框架无构建
- ✅ **移动优先** — 以 375-414px 为设计基准
- ✅ **PWA 就绪** — manifest.json + service worker 骨架
- ✅ **数据持久化** — 全部使用 localStorage
- ✅ **设计统一** — CSS Variables 驱动 ZA Bank 风格
- ❌ **无响应模式** — 图片为 base64，存储压力大
- ❌ **无网络层** — 纯本地，无服务器/无同步
- ❌ **模块耦合** — 模块间通过 `PhotoModule.showToast` 等全局引用，无事件总线

## 二、深度痛点挖掘

### 2.1 🤔 女生视角（最终用户）痛点

#### 📸 拍照打卡

| # | 痛点 | 严重程度 | 说明 |
|---|------|----------|------|
| P1 | **打卡记录不可回顾** | 🔴 高 | 拍完照片只在 localStorage 存 base64，界面上看不到历史照片墙，女生无法回顾"之前的自己" |
| P2 | **打卡激励不足** | 🔴 高 | 连续打卡没有里程碑反馈（7天/14天/30天），缺少持续使用的动力 |
| P3 | **拍照流程卡顿** | 🟡 中 | 按钮状态切换过多（"开始拍照"→"打开相机中"→"咔嚓拍照"→"再拍一张"），流程不够丝滑 |
| P4 | **彩虹屁可预期** | 🟡 中 | 模板虽多但结构相似（`${w}的你${m}`），时间长了女生会觉得"又是这个套路" |
| P5 | **提醒时间固定** | 🟢 低 | 只在 11:30-13:00 触发，但某些上班族/学生党中午不方便自拍 |

#### 🍽️ 午餐大转盘

| # | 痛点 | 严重程度 | 说明 |
|---|------|----------|------|
| P1 | **餐厅非真实数据** | 🔴 高 | 预置的"饺子馆、兰州拉面、轻食沙拉"是虚拟数据，没有接入真实地理信息，女生会觉得"不可信" |
| P2 | **选择后无后续** | 🟡 中 | 转盘结果只是一个弹窗，没有"查看详情""导航去""收藏"等下一步操作 |
| P3 | **缺少排斥机制** | 🟡 中 | 今天不想吃某家时只能手动删除，不能"跳过这个/重新转" |
| P4 | **历史统计缺失** | 🟢 低 | 只展示最近5条历史，没有"最常吃""本周吃了几次"等有趣统计 |
| P5 | **转盘缺乏惊喜感** | 🟢 低 | 每次转盘体验一样，缺少特效、声音、抖动等感官反馈 |

#### ✨ 愿望池

| # | 痛点 | 严重程度 | 说明 |
|---|------|----------|------|
| P1 | **男朋友不在场** | 🔴 高 | **核心痛点** — 女生许了愿/吐了槽，男朋友根本看不到！这是双向情感连接最大的断裂点 |
| P2 | **气泡布局拥挤** | 🟡 中 | 愿望一多，气泡杂乱堆叠，没有分类/筛选，使用体验下降 |
| P3 | **吐槽与愿望视觉区分弱** | 🟡 中 | 吐槽和愿望只在文字前加 emoji，气泡样式上区分不够明显 |
| P4 | **缺少已实现标记** | 🟢 低 | 男朋友帮实现了愿望（如买了奶茶），女生这边无法标记"已实现" |

#### 🧭 整体体验

| # | 痛点 | 严重程度 | 说明 |
|---|------|----------|------|
| P1 | **缺少"我们"的维度** | 🔴 高 | 整个 App 只有女生在玩，男朋友完全缺席。纪念日、在一起天数、爱的数据等"两个人"的元素缺失 |
| P2 | **缺少首页/仪表盘** | 🟡 中 | 没有"今天摘要"——今天拍了没？午餐选了啥？收到什么留言？进入 App 没有明确的第一落点 |
| P3 | **情感连接单向** | 🔴 高 | 男朋友可以送 App 给女朋友，但之后无法参与、无法互动、无法回应 |
| P4 | **数据零安全感** | 🟡 中 | localStorage 可能被清除、换手机丢失数据，所有爱的记录可能一夜消失 |
| P5 | **无个性化** | 🟢 低 | 没有女朋友的名字、男朋友的名字、纪念日配置，通用感强，专属感弱 |

### 2.2 现有 UI 问题诊断

| 问题 | 说明 |
|------|------|
| 情感温度不足 | Za Bank 风格简洁但偏冷淡（银行APP风格），需要升温到"恋人之间"的温度 |
| 页面层级扁平 | 三个 tab 并列，没有信息优先级，缺少信息架构 |
| 空白状态简陋 | 愿望池空时只有文字提示，缺乏情感化插画 |
| 动效单一 | 只有 fadeIn 和气泡 bounce-in，转场缺少连贯性 |
| 文字排版平淡 | 全局使用相同字体层级，标题/正文/标注视觉区分不够 |
| 缺乏微交互 | 按钮点击、tab 切换、转盘旋转缺少触感/声反馈 |

### 一、功能设计

#### 3.1.1 新增功能（WHY + 解决什么痛点）

##### 🚀 P0 — 必须实现

| # | 功能 | 解决痛点 | 方案简述 |
|---|------|----------|----------|
| F-01 | **💑 首页仪表盘（今天摘要）** | P2-整体-缺少首页 | 进入 App 的第一页，展示：在一起天数、今日打卡状态、今日午餐选择、最新愿望气泡、男朋友最新留言。三个功能页作为子页 |
| F-02 | **📸 照片墙 / 记忆画廊** | P1-拍照-不可回顾 | 在拍照打卡页增加"过去照片"区域，以缩略图网格展示最近 7 天打卡照片，点击可全屏浏览+查看当时的彩虹屁 |
| F-03 | **📝 男朋友留言板** | P1-愿望-男朋友不在场、P3-整体-情感单向 | 让男朋友可以（通过 localStorage 写入或简单管理界面）预先写留言，女生在愿望池tab里可以看到"来自他的话"。留言支持：早安、晚安、随机惊喜、特殊日触发 |
| F-04 | **❤️ 纪念日与爱的数据** | P1-整体-缺少"我们"的维度 | 在首页顶部展示：在一起 X 天、累计打卡 X 天、累计转盘 X 次、愿望实现 X 个。设置页面可配置纪念日日期 |
| F-05 | **🏆 打卡成就系统** | P2-拍照-激励不足 | 连续打卡触发成就徽章：7天（萌芽❤️）、14天（绽放🌸）、21天（盛放🌺）、30天（永恒💎）。每次达成触发庆祝动效 |

##### 🚀 P1 — 重要

| # | 功能 | 解决痛点 | 方案简述 |
|---|------|----------|----------|
| F-06 | **🎯 午餐结果增强** | P2-午餐-选择后无后续 | 转盘结果弹窗增加：查看地图导航按钮（接入高德/百度地图URI）、收藏该餐厅、标记为"今天不想吃"排除 |
| F-07 | **🔁 排除重转** | P3-午餐-缺少排斥机制 | 转盘结果增加"不吃这个，再转一次"按钮，排除当前选中项重新转 |
| F-08 | **📊 午餐统计** | P4-午餐-缺少历史统计 | 增加"午餐周报"：本周选中次数TOP3、各餐厅被选次数、本周吃了几次转盘 |
| F-09 | **✨ 愿望池升级** | P2-愿望-气泡拥挤、P3-愿望-区分弱 | 愿望和吐槽使用不同气泡样式（愿望：柔和星星形/圆形渐变；吐槽：带小闪电或四边形），增加"仅看愿望""仅看吐槽"筛选 |
| F-10 | **✅ 愿望已实现标记** | P4-愿望-缺少已实现标记 | 女生手动标记愿望已实现（或男朋友标记），已实现愿望变为金色 + 完成勾，沉底展示 |
| F-11 | **🗂️ 数据备份/导出** | P4-整体-数据安全感 | 导出所有数据为 JSON（打卡记录 + 愿望 + 历史），可导入恢复。给女生一个"下载备份"按钮 |

##### 🚀 P2 — 锦上添花

| # | 功能 | 解决痛点 | 方案简述 |
|---|------|----------|----------|
| F-12 | **🌈 彩虹屁进阶** | P4-拍照-彩虹屁可预期 | 增加基于真实照片的分析（若技术可行）或更多样化的夸夸结构，加入"男朋友口吻"的更真实夸奖（如"今天这个口红颜色很适合你"） |
| F-13 | **⏰ 提醒自定义** | P5-拍照-提醒时间固定 | 女生可选择提醒时间：中午/下午/傍晚/早上，或自定义时间 |
| F-14 | **🎮 小游戏彩蛋** | 增加惊喜感 | 在某个角落藏一个简易小游戏（戳泡泡/翻牌），增加趣味性 |
| F-15 | **📱 情侣同屏模式设想** | 长期机会 | 远景：通过 WebRTC 或简单后端实现两人共享——男朋友也能收到"她打卡了"的通知 |

#### 3.1.2 现有功能优化

| 模块 | 优化项 | 说明 |
|------|--------|------|
| 拍照 | 拍照流程简化 | 一次点击打开相机并自动预览 → 点击拍照 → 自动生成彩虹屁，减少中间状态，使用 Promise 链式处理 |
| 拍照 | 连拍逻辑 | 支持连按连拍（连续拍照不关闭摄像头），拍完选择"用哪张" |
| 拍照 | 照片存储优化 | base64 压缩 + 限制最多存 14 天（而非30天），旧的自动清除 |
| 午餐 | 转盘音效/触感 | iOS/Android 原生触感反馈，CSS 震动动画 |
| 午餐 | 默认餐厅本地化 | 调用高德/百度地图 API 获取用户附近真实餐厅（之前有设计但未实现） |
| 愿望池 | 删除二次确认 | 清空愿望时增加确认弹窗，避免误触 |
| 全局 | Toast 升级 | 区分成功/失败/信息三种 Toast 样式，增加图标和颜色区分 |
| 全局 | 加载状态优化 | 各页面按需加载骨架屏，减少白屏感 |

### 二、UI设计

#### 3.2.1 设计风格方向

**延续 + 进化：Za Bank 2.0 — 温暖恋人版**

| 维度 | v1.0（ZA Bank） | v2.0（温暖恋人版） |
|------|----------------|-------------------|
| 情感基调 | 简洁、干净、可爱、偏产品感 | 温暖、甜蜜、私人感、细腻 |
| 色彩 | 粉紫高饱和渐变 | 加入暖杏色、奶油色底色，粉紫色作为点缀色 |
| 圆角 | 统一 16px/24px | 更大圆角（20px/28px），更 soft |
| 卡片 | 白底+细边框 | 加入微妙的渐变色底+柔和阴影，更立体 |
| 字重 | 600/700 为主 | 引入 400/500 更轻盈的正文字重 |
| 装饰 | 简单圆点装饰线 | 加入手绘风格的心形、星星、小花装饰元素 |
| 留白 | 标准留白 | 增加情感留白，呼吸感更强 |
| 插图 | 无 | 加入情感化小插画（SVG 极简风格） |

#### 3.3.2 动效交互建议

##### 转场动画

| 场景 | 动画 | 实现方案 |
|------|------|----------|
| Tab 切换 | 页面横向滑入滑出 | CSS `translateX` 动画，250ms ease-out |
| 子页进入 | 从右向左滑入 | CSS `translateX(100%) → translateX(0)` |
| 子页返回 | 从左向右滑出 | 逆向动画 |
| 首页天数数字 | 数字滚动翻牌 | setInterval 计数 + transform scale 脉冲 |
| 弹窗出现 | 缩放弹入 + 背景渐变 | `scale(0.8) → scale(1)` + backdrop blur 渐入 |
| 弹窗关闭 | 缩小淡出 | `scale(1) → scale(0.9) + opacity: 0` |

##### 微交互动效

| 元素 | 动效 | 触发条件 |
|------|------|----------|
| 底部导航图标 | 弹跳 + 放大 | 选中时 `scale(1) → scale(1.15) → scale(1)` |
| 按钮点击 | iOS 风格压感 | `scale(0.96)` + 轻微阴影缩小 |
| Toggle 开关 | 滑块平滑滑动 | `translateX(0) → translateX(20px)` 300ms |
| 转盘指针 | 弹簧回弹 | 转盘停止后指针 overshoot + 回弹 |
| 气泡出现 | 弹入 + 漂浮动 | 每个气泡随机 delay 0-200ms 依次弹入 |
| 彩虹屁文字 | 打字效果 | 逐字出现（可选开关，增加仪式感） |
| 成就达成 | 庆祝彩带 + 徽章放大 | CSS keyframes confetti 效果 |
| 打卡成功 | 心跳脉冲 | 照片周围光圈 pulse 动画 |

##### 触感反馈（Haptics）

| 操作 | 触感 | 实现 |
|------|------|------|
| Tab 切换 | 轻触 (light impact) | `navigator.vibrate(10)` |
| 拍照 | 快门触感 | `navigator.vibrate([15, 30, 15])` 短震 |
| 转盘停止 | 提示触感 | `navigator.vibrate(50)` |
| 提交愿望 | 轻触确认 | `navigator.vibrate(20)` |
| 成就达成 | 双重震动 | `navigator.vibrate([50, 100, 50])` |
| 错误操作 | 警告触感 | `navigator.vibrate([30, 50, 30])` |

> ⚠️ 降级处理：对于不支持 `navigator.vibrate` 的设备（如 iOS Safari），使用 CSS 动画替代触感反馈

#### 3.3.4 移动端手势交互

| 手势 | 区域 | 操作 | 说明 |
|------|------|------|------|
| ← 右滑 | 页面左边沿 | 返回上一页 | 从子页返回首页/从全屏照片返回 |
| → 左滑 | 页面右边沿 | 无（预留） | 预留未来扩展 |
| ↓ 下滑 | 页面顶部 | 刷新数据 | 重新加载当天数据（暂不实现下拉刷新） |
| ←→ 左右滑动 | 照片墙全屏模式 | 切换上一张/下一张照片 | 触摸滑动切换历史照片 |

- 所有可点击元素 `cursor: pointer` + `active: scale(0.96)`
- 滑动切换时手指跟随 + 松手自动对齐（snap scrolling）
- 照片墙使用 CSS `scroll-snap-type: x mandatory`

### 五、设计总结

**核心设计理念：**

> 「从一个人的工具，变成两个人的连接」

v2.0 迭代不是简单加功能，而是重新定义了产品定位：

| v1.0 | v2.0 |
|------|------|
| 📸 她自拍 → 他看彩虹屁 | 📸 她自拍 → 他夸 → 她看到他的回应（留言板） |
| 🍽️ 她一个人决定午餐 | 🍽️ 他也能推荐餐厅、帮她做决定（通过留言板） |
| ✨ 她的愿望他看不见 | ✨ 她的愿望他回应 → 实现标记 → 双向确认 |
| 三个独立功能 | 首页串联三个功能 + 展示"我们"的整体状态 |

**三个设计关键词：**
1. **温暖** 🧡 — 配色、字体、圆角、留白全面升温
2. **连接** 💞 — 留言板、已实现标记、爱的数据，让两个人都在场
3. **持续** 🌱 — 成就系统、照片墙、连续打卡，让爱在每一天积累
